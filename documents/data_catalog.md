# Data Catalog – Gold Layer

## Overview

The **Gold Layer** contains business-ready data modeled for reporting, business intelligence (BI), dashboards, and analytical workloads. It follows a **star schema** consisting of dimension tables and a central fact table.

### Data Model

| Table | Type | Description |
|--------|------|-------------|
| **gold.dim_customers** | Dimension | Stores customer master data enriched with demographic and geographic information. |
| **gold.dim_products** | Dimension | Stores product master data including category, subcategory, and product attributes. |
| **gold.fact_sales** | Fact | Stores sales transactions linked to customers and products for analytical reporting. |

---

# 1. gold.dim_customers

## Purpose

Stores customer information enriched with demographic and geographic attributes for customer analytics.

## Table Grain

**One record per customer.**

## Primary Key

`customer_key`

| Column | Data Type | Description |
|---------|-----------|-------------|
| customer_key | INT | Surrogate key uniquely identifying each customer record. |
| customer_id | INT | Original and unique customer identifier from the source system. |
| customer_number | NVARCHAR(50) | Alphanumeric identifier representing the customer, used for tracking and referencing.. |
| first_name | NVARCHAR(50) | Customer's first name. |
| last_name | NVARCHAR(50) | Customer's last name. |
| country | NVARCHAR(50) | Customer's country of residence (e.g., 'Australia'). |
| marital_status | NVARCHAR(50) | Customer's marital status (e.g., 'Married','Single'). |
| gender | NVARCHAR(50) | Customer's gender (e.g., 'Male', 'Female', 'n/a'). |
| birthdate | DATE | Customer's date of birth formatted as YYYY-MM-DD (e.g., 1971-10-06). |
| create_date | DATE | Date the customer record was created in the  system. |

---

# 2. gold.dim_products

## Purpose

Stores product information and descriptive attributes used for product analytics.

## Table Grain

**One record per active product.**

## Primary Key

`product_key`

| Column | Data Type | Description |
|---------|-----------|-------------|
| product_key | INT | Surrogate key uniquely identifying each product record. |
| product_id | INT | Original product identifier from the source system. |
| product_number | NVARCHAR(50) | Alphanumeric business identifier used to uniquely identify and track the product in the source system.. |
| product_name | NVARCHAR(50) |Descriptive product name that may include attributes such as model, color, size, or style. |
| category_id | NVARCHAR(50) | Identifier of the product category. |
| category | NVARCHAR(50) | High-level product category. |
| subcategory | NVARCHAR(50) | Product subcategory. |
| maintenance | NVARCHAR(50) | Indicates whether the product requires maintenance (e.g., 'Yes', 'No'). |
| cost | INT | Standard cost of the product measured in monetary units. |
| product_line | NVARCHAR(50) | Product line classification (e.g., Road, Mountain). |
| start_date | DATE | Date the product became active. |

---

# 3. gold.fact_sales

## Purpose

Stores sales transactions used to analyze revenue, customer purchasing behavior, product performance, and sales trends.

## Table Grain

**One record per sales order line.**

## Primary Key

This fact table has **no primary key**, as multiple products can belong to the same order.

## Foreign Keys

- `customer_key` → `gold.dim_customers.customer_key`
- `product_key` → `gold.dim_products.product_key`

| Column | Data Type | Description |
|---------|-----------|-------------|
| order_number | NVARCHAR(50) | Alphanumeric identifier used to uniquely identify and track each sales order (e.g., 'SO54496'). |
| product_key | INT | Surrogate key referencing the associated product record in the product dimension table. |
| customer_key | INT | Surrogate key referencing the associated customer record in the customer dimension table. |
| order_date | DATE | Date the order was placed. |
| shipping_date | DATE | Date the order was shipped. |
| due_date | DATE | Date payment was due. |
| sales_amount | INT | Revenue generated by the sales order line in whole currency units (e.g., 95). |
| quantity | INT | Number of product units sold (e.g., 1). |
| price | INT | Selling price per unit of the product (e.g., 65). |

# Notes

- **Dimension tables** provide descriptive attributes used to filter, group, and categorize business data.
- **Fact tables** store measurable business events such as sales transactions.
- Surrogate keys (`customer_key` and `product_key`) are generated within the data warehouse and are used to establish relationships between dimensions and the fact table.
- The Gold layer is optimized for reporting, dashboarding, SQL analytics, and downstream Business Intelligence (BI) tools such as Power BI and Tableau.
